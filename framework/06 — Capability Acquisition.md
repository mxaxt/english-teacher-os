# **Capítulo 6 — Capability Acquisition**

## **Cómo se construye una capacidad**

**Versión:** 1.0  
**Estado:** Aprobado

---

# **Introducción**

Hasta ahora hemos definido qué es una capacidad.

Todavía no hemos explicado cómo nace.

Éste es uno de los problemas más importantes del framework.

Porque un profesor no crea capacidades.

Las capacidades se construyen dentro del estudiante.

El profesor únicamente diseña las condiciones para que eso ocurra.

Este capítulo describe ese proceso.

---

# **1\. El error del modelo tradicional**

La mayoría de los cursos supone un proceso parecido al siguiente.

Explicación

↓

Ejercicio

↓

Corrección

↓

Aprendizaje

Nuestro framework considera que este modelo es insuficiente.

No explica por qué muchas personas aprenden una regla y siguen sin poder utilizarla.

---

# **2\. Nuestra hipótesis**

Proponemos un ciclo diferente.

Experiencia

↓

Intento

↓

Feedback

↓

Ajuste

↓

Nuevo intento

↓

Automatización

↓

Transferencia

Este ciclo aparece constantemente.

No ocurre una sola vez.

---

# **3\. Todo comienza con un problema**

El cerebro aprende cuando necesita resolver algo.

No cuando recibe información.

Por eso cada experiencia debe contener un objetivo comunicativo real.

Ejemplos.

Contar una historia.

Convencer.

Pedir ayuda.

Resolver un conflicto.

Dar una opinión.

El conocimiento aparece después.

Nunca antes.

---

# **4\. El primer intento**

El estudiante siempre intenta resolver el problema utilizando las herramientas que ya posee.

Aunque sean insuficientes.

Este principio es fundamental.

Nunca mostramos inmediatamente la solución.

Primero necesitamos observar.

¿Qué sabe hacer?

¿Qué intenta hacer?

¿Qué evita hacer?

El intento revela mucho más que cualquier examen.

---

# **5\. El valor del error**

Cuando el estudiante falla ocurren dos cosas.

Primero.

Obtiene información sobre el problema.

Segundo.

El profesor obtiene información sobre el estudiante.

Por eso los errores benefician simultáneamente al alumno y al sistema.

Nunca deben eliminarse completamente.

---

# **6\. El feedback**

El feedback tiene un único objetivo.

Reducir la distancia entre el comportamiento actual y el comportamiento deseado.

No pretende demostrar autoridad.

No pretende señalar fallas.

Pretende facilitar el siguiente intento.

Por ello el feedback siempre apunta hacia adelante.

Nunca hacia atrás.

---

# **7\. El ajuste**

Después del feedback aparece una etapa silenciosa.

El estudiante reorganiza internamente su modelo.

No siempre es visible.

No siempre ocurre inmediatamente.

Pero constituye el verdadero aprendizaje.

El profesor no puede producir este cambio.

Únicamente puede favorecerlo.

---

# **8\. El segundo intento**

Éste es uno de los momentos más importantes del framework.

Nunca asumimos que una explicación produjo aprendizaje.

Necesitamos observar un nuevo intento.

Si el comportamiento mejora.

Aumenta nuestra confianza.

Si no mejora.

La explicación probablemente fue insuficiente.

---

# **9\. Repetición con variación**

Toda capacidad necesita repetirse.

Pero nunca exactamente igual.

El contexto cambia.

El objetivo permanece.

Ejemplo.

Hoy.

Contar un viaje.

Mañana.

Contar una experiencia laboral.

Después.

Contar una anécdota divertida.

La estructura cognitiva es la misma.

La situación cambia.

Así aparece la transferencia.

---

# **10\. Consolidación**

Después de múltiples intentos exitosos.

La capacidad comienza a estabilizarse.

El estudiante necesita menos tiempo.

Menos esfuerzo.

Menos autocorrecciones.

El sistema interpreta estas señales como evidencia de consolidación.

---

# **11\. Automatización**

La automatización no aparece porque pasó tiempo.

Aparece porque el comportamiento fue utilizado repetidamente en contextos variados.

El estudiante deja de recuperar reglas.

Empieza a recuperar soluciones.

Ese cambio representa uno de los principales objetivos del framework.

---

# **12\. Transferencia**

Una capacidad verdaderamente adquirida aparece en contextos nuevos.

Si el estudiante sólo puede utilizarla dentro del ejercicio original.

Todavía no está consolidada.

El sistema buscará continuamente oportunidades inesperadas para verificar la transferencia.

---

# **13\. Integración**

Las capacidades nunca permanecen aisladas.

Cada nueva adquisición modifica otras capacidades existentes.

Por ejemplo.

Aprender a expresar opiniones mejora.

Debates.

Conversaciones.

Presentaciones.

Entrevistas.

El sistema intenta aprovechar constantemente estas sinergias.

---

# **14\. Olvido**

Olvidar no constituye un fracaso.

Forma parte del aprendizaje.

Cuando una capacidad comienza a debilitarse.

El sistema no reinicia el proceso.

Simplemente reactiva la red correspondiente.

El reaprendizaje suele ser mucho más rápido que el aprendizaje inicial.

---

# **15\. El papel del tiempo**

El tiempo, por sí solo, no produce aprendizaje.

La calidad de las experiencias sí.

Diez conversaciones superficiales pueden producir menos crecimiento que una conversación cuidadosamente diseñada.

El sistema optimiza experiencias.

No horas de estudio.

---

# **16\. Evidencia de adquisición**

Consideramos que una capacidad está siendo adquirida cuando observamos.

Mayor velocidad.

Menor esfuerzo.

Mayor estabilidad.

Mayor flexibilidad.

Mayor transferencia.

Menor necesidad de corrección.

Ninguna de estas señales resulta suficiente por separado.

Juntas forman evidencia sólida.

---

# **17\. El algoritmo de adquisición**

Toda capacidad sigue el mismo ciclo.

Presentar un desafío.

↓

Observar el intento.

↓

Interpretar el comportamiento.

↓

Elegir la intervención mínima.

↓

Generar un nuevo intento.

↓

Actualizar el Student Model.

↓

Programar futuras reactivaciones.

↓

Buscar evidencia de transferencia.

El Learning Engine ejecuta este algoritmo continuamente.

---

# **18\. Principio fundamental**

Las capacidades no se enseñan.

Se construyen.

El profesor no introduce habilidades dentro del estudiante.

Diseña una secuencia de experiencias que permiten al estudiante construirlas por sí mismo.

Esta diferencia define todo el framework.

---

# **Conclusión**

El AI Learning Framework entiende el aprendizaje como un proceso iterativo de construcción.

Cada experiencia modifica ligeramente el sistema cognitivo del estudiante.

Cada modificación facilita la siguiente.

Miles de pequeños cambios producen grandes transformaciones.

La función del profesor no consiste en acelerar artificialmente ese proceso.

Consiste en diseñar las mejores condiciones posibles para que ocurra.

Y la función de la IA es hacer que esas condiciones sean únicas para cada estudiante, adaptándolas continuamente a su evolución.
